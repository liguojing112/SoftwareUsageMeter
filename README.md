# SoftwareUsageMeter

## 2026-04-25 交付补充

本次补充来自第二次现场交付后的复盘：雇主电脑上缺少 Visual C++ 运行库，导致程序在导出检测 / 原生依赖加载时表现异常，误看起来像“收费框无法触发”。当前打包配置已把常用 Visual C++ 运行时 DLL 一起打进 exe，后续交付必须使用最新 `build.bat` / `build.spec` 打包产物。

- `build.spec` 已打包 `msvcp140*.dll`、`vcruntime140*.dll`、`concrt140.dll` 等运行库文件，降低目标电脑未安装 Visual C++ Redistributable 时的崩溃风险。
- 启动 `SoftwareUsageMeter.exe` 后会先弹出操作说明提示框，现场人员必须阅读并点击“知道了”后，程序才开始进入监控流程。
- 启动提示框会明确说明：点击像素蛋糕导出按钮后，先弹出导出框，再弹出收费框；客户付款后输入管理员密码解锁，再让客户继续导出照片。
- 顾客点击像素蛋糕右上角黄色“导出 / 导出至本地”按钮后，程序会尽快弹出“请等待 5 秒钟”的全屏等待遮罩，用来阻止重复点击，并给收费框检测争取时间。
- 等待遮罩属于正常保护流程，不代表程序卡死；5 秒后如果已确认进入导出流程，会继续显示正式收费弹窗。
- 如果现场出现启动异常、无日志、收费框不弹，优先确认雇主使用的是最新 `dist\SoftwareUsageMeter.exe`，不要混用旧 exe 或旧打包目录。

## 2026-04-24 交付前更新

本次版本围绕现场交付稳定性做了补充，已在另一台 Windows 电脑上做过兼容性测试，未发现明显异常。

- 启动 `SoftwareUsageMeter.exe` 后会弹出友好提示窗，提示现场人员按正确流程操作，并等待点击“知道了”。
- 启动提示窗支持使用管理后台配置的壁纸；未设置壁纸时会使用浅色兜底背景。
- 程序不会在像素蛋糕未真正打开主窗口时提前计时，避免残留进程或后台进程导致误计时。
- 关闭像素蛋糕后，本轮计时会结束并清零；再次打开像素蛋糕会从 0 开始新的计时周期。
- 管理员确认收款后会执行更稳的交互恢复，避免像素蛋糕窗口残留锁定或半挂起。
- 付款后的同一次导出尾迹会被保护，避免图片真正导出时再次弹出收费框。
- exe 运行日志写在 exe 同目录的 `app.log`；启动早期异常还会尝试写入 `%TEMP%\SoftwareUsageMeter_bootstrap.log`。

## 当前版本功能

- 自动监控 `PixCake.exe` 是否启动 / 退出
- 自动计时，按分钟计费
- 导出时弹出全屏置顶收费窗口
- 导出按钮点击后先显示 5 秒全屏等待遮罩，防止顾客重复点击或未付款先导出
- 计费公式支持：
  - 使用时长费用
  - 导出张数费用
- 自动识别导出张数，识别失败时使用默认值兜底
- 支持普通导出页与“导出至本地”对话框
- 支持管理员密码确认收款
- 支持分别配置微信 / 支付宝收款码
- 支持管理页、密码页、状态页、收费页壁纸
- 打包版随带常用 Visual C++ 运行库 DLL，提升雇主电脑兼容性
- `app.log` 支持现场排障日志

## 计费公式

```text
总费用 = 使用时长（分钟）× 计时单价 + 导出张数 × 单张导出单价
```

## 快速开始

### 运行源码

```powershell
pip install -r requirements.txt
python main.py
```

### 打包

```powershell
build.bat
```

或：

```powershell
pyinstaller build.spec --clean --noconfirm
```

> 交付必须使用最新打包脚本。`build.spec` 会把 Visual C++ 运行库 DLL 一起打包，避免目标电脑未安装 VC++ 运行库时出现启动异常或导出检测异常。

### 运行 exe

```text
dist\SoftwareUsageMeter.exe
```

## 首次配置

1. 启动程序
2. 右键托盘图标，打开管理设置
3. 输入管理员密码
4. 配置：
   - 计时单价
   - 单张导出单价
   - 微信收款码
   - 支付宝收款码
   - 壁纸
   - 目标进程名
5. 保存设置

默认管理员密码：`admin`

## 日志说明

程序运行后会在源码目录或 exe 同目录生成：

```text
app.log
```

日志里会记录：

- 运行环境
- 当前配置摘要
- 监控器初始化参数
- 目标程序启动 / 退出
- 导出检测诊断
- 收费窗口触发原因
- 导出结束 / 付款确认

如果现场再次出现“收费框没有弹出”，优先收集 `app.log`。

## 交付前建议测试

- 在目标电脑上只运行最新 `dist\SoftwareUsageMeter.exe`，不要混用旧版本 exe
- 首次启动能看到操作说明提示框，并且点击“知道了”后程序进入托盘运行
- 点击像素蛋糕右上角黄色导出按钮后，能看到“请等待 5 秒钟”全屏遮罩
- 普通编辑页不弹收费框
- 普通导出页弹收费框
- “导出至本地”对话框弹收费框
- 管理员确认收款后可继续导出
- 退出程序后像素蛋糕仍可正常操作
- exe 同目录能生成 `app.log`
