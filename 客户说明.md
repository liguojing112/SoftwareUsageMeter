# SoftwareUsageMeter 客户说明

## 2026-04-24 交付版补充说明

本次交付版重点加强了启动提示、计时周期和现场异常恢复。已在另一台 Windows 电脑上做兼容性测试，未发现明显异常。

### 启动后的提示

双击 `SoftwareUsageMeter.exe` 后，程序会弹出一个友好的启动提示窗，提示软件已在后台运行，并正在等待像素蛋糕启动。这个提示窗会自动关闭，也可以点击“知道了”手动关闭。

提示窗会使用管理后台设置的壁纸。如果没有设置壁纸，则显示默认浅色背景。

### 计时规则调整

程序现在只在检测到像素蛋糕真实主窗口后开始计时。仅有后台残留进程、最小化异常窗口或过渡进程时，不会提前开始计费。

如果顾客中途关闭像素蛋糕，本轮使用会直接结束并清零。顾客再次打开像素蛋糕时，会从 0 开始新的计时周期。

### 收款后的恢复

管理员输入密码并确认收款后，程序会自动恢复像素蛋糕窗口交互，避免付款后窗口仍被锁住。付款后的同一次导出过程不会再次弹出收费框。

### 现场排查

如果现场出现“没有弹收费框”“程序启动异常”“状态页空白”等问题，优先收集 exe 同目录下的 `app.log`。如果 `app.log` 没有生成，再查看 `%TEMP%\SoftwareUsageMeter_bootstrap.log`。

本文档面向门店操作人员，说明软件用途、使用流程、后台设置与常见问题处理。

## 1. 软件用途

本软件配合“像素蛋糕”使用，用于门店自助修图场景的自动计时与收费管理。

工作流程如下：

1. 顾客打开像素蛋糕开始修图
2. 程序自动开始计时
3. 顾客进入导出流程时，程序弹出收费窗口
4. 管理员确认收款后，顾客再继续导出

---

## 2. 计费方式

程序采用混合计费：

```text
总费用 = 使用时长（分钟）× 计时单价 + 导出张数 × 单张导出单价
```

示例：

- 使用 5 分钟
- 计时单价 2 元 / 分钟
- 导出 3 张
- 单张导出单价 4 元 / 张

则总费用为：

```text
5 × 2 + 3 × 4 = 22 元
```

## 3. 主要功能

### 3.1 自动计时

- 打开像素蛋糕后自动开始计时
- 关闭像素蛋糕后自动暂停计时

### 3.2 自动检测导出

程序会自动识别以下场景：

- 普通导出页
- “导出至本地”对话框
- 创意流程中进入“导出至本地”的对话框

### 3.3 收费弹窗

收费弹窗会显示：

- 使用时长
- 计时单价
- 导出张数
- 单张导出单价
- 计时费用
- 导出费用
- 合计金额
- 微信收款码
- 支付宝收款码

### 3.4 管理员确认

- 顾客不能直接关闭收费窗口
- 点击“已付款 · 确认收款”后需要输入管理员密码
- 密码正确后才会关闭收费窗口并恢复导出

## 4. 首次使用

### 4.1 打开管理设置

1. 启动程序
2. 右键托盘图标
3. 选择“打开管理面板”
4. 输入管理员密码

默认密码：`admin`

> 建议首次使用后立即修改管理员密码。

### 4.2 需要配置的内容

建议优先配置：

- 计时单价
- 单张导出单价
- 微信收款码
- 支付宝收款码
- 壁纸
- 目标进程名（默认 `PixCake.exe`）

## 5. 日常使用流程

### 场景一：普通修图后导出

1. 顾客打开像素蛋糕开始修图
2. 程序自动计时
3. 顾客进入导出流程
4. 程序弹出收费窗口
5. 顾客扫码付款
6. 管理员点击“已付款 · 确认收款”
7. 输入管理员密码
8. 顾客继续导出

### 场景二：导出至本地

1. 顾客点击“导出至本地”
2. 本地导出对话框出现
3. 程序弹出收费窗口
4. 管理员确认收款
5. 顾客继续完成导出

## 6. 导出张数说明

程序会自动识别导出张数。  
如果 OCR 识别失败，会使用后台设置中的默认导出张数作为兜底值。

这不会影响收费流程是否弹出，只会影响导出张数的取值。

## 8. 常见问题

### Q1：为什么收费框没有弹出？

请先检查：

1. 像素蛋糕是否正常运行
2. 是否已经进入导出流程
3. 后台目标进程名是否为 `PixCake.exe`
4. 程序是否正常运行在托盘中

如仍有问题，请把 `app.log` 发给开发人员。

### Q2：为什么导出张数不准确？

可能原因：

- OCR 识别失败
- 导出对话框未完全显示
- 界面被遮挡

可以先在后台设置中调整默认导出张数作为兜底。

### Q3：为什么退出程序后像素蛋糕无法操作？

当前版本已加入窗口恢复机制。  
如果偶发异常，请重新启动本程序一次，一般会自动恢复像素蛋糕窗口状态。

### Q4：日志文件在哪里？

运行源码时在项目目录。  
运行 exe 时在 exe 同目录。

文件名：

```text
app.log
```
