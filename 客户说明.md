# SoftwareUsageMeter 客户说明

## 2026-04-26 导出保护与张数确认说明

当前版本把导出前保护拆成两步：程序启动时先准备好轻量守卫，顾客真正点击导出按钮时再显示等待遮罩和收费框。

- 程序启动后会提前预热“请等待 7 秒钟”遮罩，并在后台轻量监听像素蛋糕右上角黄色“导出 / 导出至本地”按钮。
- 这个轻量守卫只是提前防乱点，不会提前收费，也不会在普通修图过程中弹收费框。
- 普通导出框里的黄色“导出”按钮同样会触发快速等待遮罩，减少慢电脑上正式收费框出现前的空档。
- 进入创意页后，只有后续真正点击“导出至本地”并进入本地导出设置框时，才会触发正式收费流程。
- 收费框会尽量读取导出框标题中的张数，例如“导出 3 张图片”。
- “导出至创意”中转页里的“原图 0 / 精修图 N”不是最终收费张数，程序会跳过这个中转页，等进入创意页后的“导出至本地”对话框再收费。
- 如果程序识别失败，或误识别成 0 张，收费框会要求管理员手动输入导出张数；未填写前不能确认收款。

## 2026-04-25 交付版补充说明

本次交付版补充了两类现场保护：

1. 目标电脑环境保护：雇主电脑曾因缺少 Visual C++ 运行库，导致程序在导出检测相关依赖加载时异常，表现为“收费框不弹”。当前最新 exe 已随包携带常用 Visual C++ 运行库文件，请务必使用最新交付的 `dist\SoftwareUsageMeter.exe`，不要混用旧版本。
2. 操作流程保护：程序增加了启动说明提示框和导出前全屏等待遮罩，避免现场人员或顾客乱点导致流程错乱。

### 启动说明提示框

双击 `SoftwareUsageMeter.exe` 后，会先弹出一个操作说明提示框。现场人员需要阅读说明，并点击“知道了”后，程序才会开始进入监控流程。

提示内容会说明正确操作顺序：

1. 顾客点击像素蛋糕导出按钮
2. 像素蛋糕弹出导出框
3. 本程序弹出收费框
4. 引导顾客扫码付款
5. 管理员输入密码确认收款
6. 解锁后再让顾客继续导出照片
7. 10秒钟后程序自动进入下一轮计时

### 导出前全屏等待遮罩

顾客点击像素蛋糕右上角黄色“导出 / 导出至本地”按钮后，程序会尽快弹出“请等待 7 秒钟”的全屏遮罩。

这是正常保护流程，作用是：

- 阻止顾客连续点击导出按钮
- 防止未付款前继续操作
- 给程序识别导出框和弹出收费框争取时间

看到“请等待 7 秒钟”时，不要强制关闭程序，也不要重复点击像素蛋糕。等待结束后，程序会继续弹出正式收费框。

## 2026-04-24 交付版补充说明

本次交付版重点加强了启动提示、计时周期和现场异常恢复。已在另一台 Windows 电脑上做兼容性测试，未发现明显异常。

### 启动后的提示

双击 `SoftwareUsageMeter.exe` 后，程序会弹出一个友好的启动提示窗。当前版本需要现场人员点击“知道了”后再进入监控流程，避免未读说明就开始操作。

提示窗会使用管理后台设置的壁纸。如果没有设置壁纸，则显示默认浅色背景。

### 计时规则调整

程序现在只在检测到像素蛋糕真实主窗口后开始计时。仅有后台残留进程、最小化异常窗口或过渡进程时，不会提前开始计费。

如果顾客中途关闭像素蛋糕，本轮使用会直接结束并清零。顾客再次打开像素蛋糕时，会从 0 开始新的计时周期。

### 收款后的恢复

管理员输入密码并确认收款后，程序会自动恢复像素蛋糕窗口交互，避免付款后窗口仍被锁住。付款后的同一次导出过程不会再次弹出收费框。

### 现场排查

如果现场出现“没有弹收费框”“程序启动异常”“状态页空白”等问题，优先收集 exe 同目录下的 `app.log`。如果 `app.log` 没有生成，再查看 `%TEMP%\SoftwareUsageMeter_bootstrap.log`。

## 1. 软件用途

本软件配合“像素蛋糕”使用，用于门店自助修图场景的自动计时与收费管理。

工作流程如下：

1. 顾客打开像素蛋糕开始修图
2. 程序自动开始计时
3. 顾客进入导出流程时，程序弹出收费窗口
4. 管理员确认收款后，顾客再继续导出

---

## 2. 计费方式

程序采用混合计费：

```text
总费用 = 使用时长（分钟）× 计时单价 + 导出张数 × 单张导出单价
```

示例：

- 使用 5 分钟
- 计时单价 2 元 / 分钟
- 导出 3 张
- 单张导出单价 4 元 / 张

则总费用为：

```text
5 × 2 + 3 × 4 = 22 元
```

## 3. 主要功能

### 3.1 自动计时

- 打开像素蛋糕后自动开始计时
- 关闭像素蛋糕后自动暂停计时

### 3.2 自动检测导出

程序会自动识别以下场景：

- 普通导出页
- “导出至本地”对话框
- 创意流程中进入“导出至本地”的对话框

### 3.3 收费弹窗

收费弹窗会显示：

- 使用时长
- 计时单价
- 导出张数
- 单张导出单价
- 计时费用
- 导出费用
- 合计金额
- 微信收款码
- 支付宝收款码

### 3.4 管理员确认

- 顾客不能直接关闭收费窗口
- 点击“已付款 · 确认收款”后需要输入管理员密码
- 密码正确后才会关闭收费窗口并恢复导出

## 5. 日常使用流程

### 场景一：普通修图后导出

1. 顾客打开像素蛋糕开始修图
2. 程序自动计时
3. 顾客点击像素蛋糕导出按钮
4. 程序可能先显示“请等待 7 秒钟”的全屏遮罩
5. 程序弹出收费窗口
6. 顾客扫码付款

2. 程序可能先显示“请等待 7 秒钟”的全屏遮罩

4. 程序弹出收费窗口
5. 管理员确认收款
6. 顾客继续完成导出

## 6. 导出张数说明

程序会自动识别导出张数。  
如果 OCR 识别失败，或识别结果为 0 张，程序不会再按默认 1 张收费，而是会在收费框中显示手动输入区域。

管理员需要根据像素蛋糕导出框里的真实张数填写数量。填写后，导出费用和合计金额会自动更新；未填写前不能确认收款。

## 8. 常见问题

### Q1：为什么收费框没有弹出？

请先检查：

1. 像素蛋糕是否正常运行
2. 是否已经进入导出流程
3. 后台目标进程名是否为 `PixCake.exe`
4. 程序是否正常运行在托盘中
5. 是否使用的是最新交付的 exe，不要混用旧版本
6. 目标电脑是否缺少 Visual C++ 运行库；当前最新版已随 exe 打包常用 VC++ 运行库文件

如仍有问题，请把 `app.log` 发给开发人员。

### Q1-1：为什么会出现“请等待 7 秒钟”？

这是程序新增的导出前保护遮罩，不是卡死。

它会在顾客点击导出按钮后出现，用来阻止重复点击和未付款前继续操作。等待结束后，如果程序确认进入导出流程，会继续显示收费框。

### Q2：为什么导出张数不准确？

可能原因：

- OCR 识别失败
- 导出对话框未完全显示
- 界面被遮挡

如果收费框提示手动输入张数，请按像素蛋糕导出框标题中的真实张数填写。当前版本不再建议依赖默认张数兜底，避免少收或多收。

### Q3：为什么退出程序后像素蛋糕无法操作？

当前版本已加入窗口恢复机制。  
如果偶发异常，请重新启动本程序一次，一般会自动恢复像素蛋糕窗口状态。

### Q4：日志文件在哪里？

运行源码时在项目目录。  
运行 exe 时在 exe 同目录。

文件名：

```text
app.log
```
