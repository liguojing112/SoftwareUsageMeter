# SoftwareUsageMeter 开发说明

本文档面向**开发者和运维人员**，介绍系统的技术架构、模块设计、配置项、调试方法和测试指南。

---

## 1. 系统概述

SoftwareUsageMeter 是运行在 Windows 门店自助设备上的计时计费工具，后台监控"像素蛋糕"（PixCake.exe）进程，在顾客进入导出流程时弹出收费框，管理员确认收款后恢复导出。

**技术栈：** Python 3.8+ / PyQt5 / psutil / pywin32 / Pillow / Windows Media OCR

---

## 2. 计费公式

```
总费用 = 使用时长（分钟）× 计时单价 + 导出张数 × 单张导出单价
```

示例：使用 5 分钟（单价 2 元/分钟），导出 3 张（单价 4 元/张）→ `5×2 + 3×4 = 22 元`

## 4. 核心模块

### 4.1 process_monitor.py —— 进程监控与 OCR

**职责：**
- 轮询检测目标进程是否启动/退出
- 捕获导出页/导出对话框截图
- 调用 Windows OCR 识别导出张数

**OCR 流程（`detect_export_image_count_from_image`）：**

1. 对截图切分多个区域变体（摘要区、分类计数区、宽范围区等）
2. 深色 UI 做 2 倍放大 + 灰度增强
3. 每个变体调用 `run_windows_ocr`（内部用 PowerShell 调用 `Windows.Media.Ocr.OcrEngine`）
4. 正则匹配数字模式（`导出\d+张`、`共\d+张` 等）
5. 多变体一致性投票：至少 2 个变体得到相同数字才认定准确
6. 失败则回退 `default_export_count`

**关键函数：**

| 函数 | 作用 |
|------|------|
| `run_windows_ocr()` | 调用 Windows 原生 OCR，返回识别文本 |
| `detect_export_image_count()` | 从窗口截图识别导出张数 |
| `extract_export_image_count_from_text()` | 从 OCR 文本正则提取数字 |
| `_prepare_export_count_ocr_variants()` | 切分图像区域生成变体 |
| `capture_window_image()` | 截取目标窗口图像 |

### 4.2 timer_manager.py —— 计时管理

**职责：**
- 管理计时状态（运行中/已暂停/已停止）
- 精确秒级计时，维护 `elapsed_seconds`
- 按配置的时间间隔触发回调

**状态机：** `idle → running ↔ paused → stopped`

### 4.3 payment_overlay.py —— 收费弹窗

**职责：**
- 全屏置顶显示收费明细（时长、单价、张数、金额、收款码）
- 锁定像素蛋糕窗口防止顾客在付款前导出
- "确认收款"按钮先验证管理员密码再关闭弹窗

**关键组件：**

| 组件 | 说明 |
|------|------|
| `_time_label` | 使用时长 |
| `_rate_label` | 计时单价 |
| `_export_count_label` | 导出张数 |
| `_export_rate_label` | 单张导出单价 |
| `_time_amount_label` | 计时费用 |
| `_export_amount_label` | 导出费用 |
| `_amount_label` | 合计金额 |
| `_confirm_btn` | "已付款·确认收款"按钮 |
| `_qr_label` | 收款码图片 |

### 4.4 admin_panel.py —— 管理后台

支持配置的参数：

| 参数 | 字段名 | 说明 |
|------|--------|------|
| 计时单价 | `rate` | 元/分钟 |
| 单张导出单价 | `export_rate` | 元/张 |
| 默认导出张数 | `default_export_count` | OCR 失败时的兜底值 |
| 管理员密码 | `admin_password` | SHA-256 哈希存储 |
| 收款码路径 | `qr_code_path` | 图片文件路径 |
| 目标进程名 | `process_name` | 默认 `PixCake.exe` |
| 导出窗口关键词 | `export_window_keywords` | 窗口识别关键词列表 |
| 监控间隔 | `monitor_interval_ms` | 毫秒，默认 2000 |

### 4.5 config_manager.py —— 配置管理

- 启动时从 `config.json` 加载配置
- 提供 `ConfigManager` 类封装所有配置项的读取与持久化
- 首次运行自动创建默认配置

  "centered_dialog_scan": true
}

## 6. 日志与调试

### 6.1 运行日志

日志文件：`app.log`

### 6.2 OCR 调试

在 `config.json` 中开启调试模式：

```json
{
  "debug_export_capture": true,
  "centered_dialog_scan": true
}
```

开启后每次导出识别会生成调试包到 `debug_export_captures/`，包含：

- 主窗口截图
- 关键裁剪图（多个变体）
- `meta.json`（窗口元数据）
- `ocr_results.json`（各变体 OCR 结果）

### 6.3 排查建议

| 问题 | 排查步骤 |
|------|----------|
| 收费框不弹出 | 检查 `app.log` 是否有 `导出摘要预判命中`、`确认导出窗口` 等关键词 |
| 导出张数不准确 | 检查调试包中关键裁剪图是否裁对、`ocr_results.json` 文本是否读到数字 |
| 退出后像素蛋糕无响应 | 重启本程序一次，启动时会自动恢复遗留的窗口锁定状态 |

## 7. 测试方法

### 7.1 自动化测试

```powershell
$env:QT_QPA_PLATFORM='offscreen'
pytest tests/ -q
```

### 7.2 手工测试清单

1. 启动程序，打开像素蛋糕，确认开始计时
2. 普通编辑/导入照片 —— 不应弹收费框
3. 普通导出页 —— 应弹收费框
4. "导出至本地"对话框 —— 应弹收费框
5. 创意模块 → "导出至本地" —— 应弹收费框
6. 管理员确认收款（需输入密码）后可继续导出
7. 付款前关闭导出页 —— 应恢复计时，不清零
8. 程序正常退出后像素蛋糕仍可操作

## 9. 架构设计要点

### 9.1 进程锁定机制

收费弹窗弹出前，通过 `win32gui.EnableWindow(hwnd, False)` 禁用像素蛋糕窗口，阻止顾客在付款前继续操作。程序退出时（正常或异常）通过 `atexit` 注册的 `_unlock_registered_windows()` 恢复所有窗口状态。

### 9.2 导出张数识别容错

采用**多变体 + 投票**策略降低误识别率：
- 同一截图生成多个不同区域的 OCR 变体
- 要求至少 2 个变体一致才认定
- 最后兜底 `default_export_count`

### 9.3 状态一致性

计时状态（`is_running`）、导出状态（`is_exporting`）、弹窗可见性（`overlay_visible`）在 `main.py` 中统一管理，确保收费流程的每一步都有明确的状态转换。

## 10. 后续开发建议

1. **响应速度**：加快"导出至本地"对话框的检测速度（当前轮询间隔 2 秒）
2. **OCR 容错**：扩大多变体覆盖范围，增强对不同分辨率/DPI 的兼容性
3. **现场恢复**：增强异常退出后的窗口状态自动恢复能力
4. **多语言支持**：扩展 OCR 引擎支持更多语言文字识别
