# SoftwareUsageMeter 开发说明

本文档面向开发者和运维人员，介绍系统架构、模块职责、配置项、日志排障与打包说明。

---

## 1. 系统概述

SoftwareUsageMeter 是运行在 Windows 门店设备上的计时计费工具，用于监控“像素蛋糕”使用过程，并在导出前弹出收费窗口。

当前核心目标：

- 监控目标进程 `PixCake.exe`
- 自动开始 / 暂停计时
- 在导出流程前触发收费弹窗
- 自动识别导出张数
- 管理员确认收款后恢复导出

技术栈：

- Python 3.11
- PyQt5
- psutil
- pywin32
- Pillow
- Windows OCR
- PyInstaller

## 3. 模块职责

### 3.1 `main.py`

职责：

- 创建 `QApplication`
- 初始化配置、计时器、监控器、收费弹窗、托盘管理器
- 连接信号
- 管理收费流程状态
- 写启动摘要日志

当前已加入：

- 监控线程延后启动
- 程序启动环境日志
- 配置摘要日志

### 3.2 `process_monitor.py`

职责：

- 监控目标进程是否运行
- 收集导出证据
- 判断是否触发收费
- 自动识别导出张数
- 写监控诊断日志

当前导出触发采用多证据并行策略，任一路命中即可触发：

| 证据类型 | 说明 |
|---------|------|
| `export_hwnd` | 导出相关窗口命中 |
| `export_worker_pid` | 检测到导出子进程 |
| `export_visual` | 识别到黄色导出按钮 |
| `export_button_clicked` | 检测到用户点击导出按钮 |
| `summary_probe` | 摘要区 OCR 命中导出张数 |
| `centered_dialog_scan` | 中央导出对话框扫描命中 |

OCR 主要负责“数张数”，不再强依赖 OCR 成功才允许弹窗。

### 3.3 `payment_overlay.py`

职责：

- 显示全屏收费弹窗
- 展示费用明细
- 展示双二维码
- 锁定导出操作直到管理员确认

当前支持：

- 微信收款码
- 支付宝收款码
- 壁纸背景
- 管理员密码确认收款

### 3.4 `admin_panel.py`

职责：

- 管理后台配置页
- 管理员密码验证页

当前支持配置：

- 计时单价
- 单张导出单价
- 目标进程名
- 导出关键字
- 微信收款码
- 支付宝收款码
- 壁纸
- 管理员密码

### 3.5 `tray_icon.py`

职责：

- 托盘图标
- 托盘菜单
- 状态页显示

当前状态页支持：

- 壁纸
- 半透明白色卡片
- 滚动条
- 计时与费用状态展示

### 3.6 `config_manager.py`

职责：

- 读取 / 保存 `config.json`
- 提供统一配置访问接口

## 4. 关键配置项

`config.json` 当前主要字段：

```json
{
  "rate": 1.0,
  "export_rate": 1.0,
  "default_export_count": 1,
  "admin_password": "sha256",
  "qr_code_path": "",
  "wechat_qr_code_path": "",
  "alipay_qr_code_path": "",
  "wallpaper_path": "",
  "process_name": "PixCake.exe",
  "export_window_keywords": ["导出", "Export"],
  "monitor_interval_ms": 2000
}
```

说明：

- `wechat_qr_code_path` / `alipay_qr_code_path`
  - 当前双码配置字段
  - 若为空，会回退到旧字段 `qr_code_path`
- `default_export_count`
  - OCR 识别失败时的兜底导出张数
- `monitor_interval_ms`
  - 基础监控周期

可选调试字段：

```json
{
  "debug_export_capture": true,
  "centered_dialog_scan": true
}
```

说明：

- `debug_export_capture`
  - 开启后保存调试截图包
- `centered_dialog_scan`
  - 控制中央导出对话框扫描兜底逻辑
  - 当前默认值为 `true`

## 5. 日志与排障

### 5.1 日志位置

- 源码运行：项目目录下的 `app.log`
- exe 运行：exe 同目录下的 `app.log`

### 5.2 当前新增的日志层次

#### 启动摘要

由 `main.py` 记录：

- `运行环境`
- `配置摘要`

可快速确认：

- 是否跑的是 exe
- 使用的 Python / 可执行文件路径
- 当前工作目录
- 壁纸和双二维码文件是否存在
- 当前费率、目标进程、关键字

#### 监控器初始化

由 `process_monitor.py` 记录：

- 目标进程名
- 导出关键字
- 监控周期
- 启动保护时间
- 视觉预热时间
- 摘要探测间隔
- 是否开启居中扫描
- 是否开启调试截图

#### 导出检测诊断

运行时每隔几秒节流记录一次，字段包括：

- `pid`
- `main_hwnd`
- `capture_hwnd`
- `capture_title`
- `capture_class`
- `export_hwnd`
- `export_worker_pid`
- `image_ok`
- `visual_candidate`
- `button_clicked`
- `summary_probe`
- `cached_count`
- `has_evidence`
- `startup_guard`
- `visual_warmup_skip`
- `dialog_mode`
- `centered_scan`
- `trigger_reason`

#### 确认触发日志

最终触发收费框时会记录：

- `确认导出窗口`
- `确认导出页预判`
- `确认导出子进程`

并携带 `trigger_reason`。

### 5.3 建议排障顺序

当现场再次出现“收费框未触发”时，按下面顺序看 `app.log`：

1. 是否出现 `运行环境`
2. 是否出现 `配置摘要`
3. 是否出现 `监控器初始化`
4. 是否出现 `检测到目标程序启动`
5. 是否出现 `导出检测诊断`
6. 是否出现：
   - `确认导出窗口`
   - `确认导出页预判`
   - `确认导出子进程`

如果第 5 步有日志但第 6 步没有，通常就能从诊断字段看出是哪条证据链没有命中。

## 6. 调试截图

如需开启导出截图调试：

```json
{
  "debug_export_capture": true
}
```

生成目录：

```text
debug_export_captures/
```

常见文件：

- `00_main.png`
- `03_summary_anchor_wide.png`
- `meta.json`
- `ocr_results.json`

排查导出识别问题时，图片和 `ocr_results.json` 需要配套看。

## 7. 打包

### 7.1 推荐方式

```powershell
build.bat
```

### 7.2 手动方式

```powershell
pyinstaller build.spec --clean --noconfirm
```

输出：

```text
dist\SoftwareUsageMeter.exe
```

### 7.3 当前打包注意点

- 打包前请关闭旧的 `SoftwareUsageMeter.exe`
- `build.bat` 已加入旧进程清理逻辑
- exe 运行日志会写到 exe 同目录，便于现场收集

## 8. 回归测试建议

### 手工回归

- 普通编辑页不弹收费框
- 普通导出页弹收费框
- “导出至本地”对话框弹收费框
- 创意流程中的“导出至本地”对话框弹收费框
- 导出张数识别正确
- 管理员确认收款后可继续导出
- 退出程序后像素蛋糕仍可正常操作
- `app.log` 在现场机器可正常生成

### 语法检查

```powershell
python -m py_compile main.py process_monitor.py admin_panel.py payment_overlay.py tray_icon.py
```
