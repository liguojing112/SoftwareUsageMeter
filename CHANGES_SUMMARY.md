# SoftwareUsageMeter 变更摘要

## 2026-04-26 启动守卫与导出张数安全更新

### 1. 启动阶段轻量守卫

**背景：** 慢电脑上收费框弹出前可能被顾客抢先点击导出，因此需要比正式 OCR 检测更早的现场保护。

**改动：**
- 程序启动阶段预热 7 秒等待遮罩，减少第一次全屏显示延迟
- 启动阶段开启轻量鼠标热区守卫，提前缓存像素蛋糕右上角黄色导出按钮位置
- 普通导出框底部黄色“导出”按钮接入独立热区守卫，减少正式检测前的空档
- 热区守卫只做鼠标和少量像素判断，不做 OCR，不影响普通修图
- 创意页“导出至本地”增加独立链路：入口不收费，进入本地导出设置框后点击黄色“导出”才收费
- 为创意页本地导出框新增标题区域裁剪，优先读取“导出 N 张图片”，不再依赖中转页的“原图 0 / 精修图 N”

### 2. 导出张数不再默认 1

**背景：** OCR 识别失败时默认 1 张可能导致少收；同时部分导出至本地场景会把“原图 0”误当成导出张数。

**改动：**
- 导出张数必须大于 0，缓存 / OCR / 摘要扫描得到 0 时全部忽略
- 对“导出 0 张图片，其中 3 张...”这类 OCR 误读增加次级标题补救
- 仍识别失败时进入 `manual_required`，收费框要求管理员手动填写张数
- 未填写有效张数前不能确认收款

## 2026-04-25 交付环境与现场保护更新

### 1. Visual C++ 运行库随包打包

**背景：** 第二次现场交付复盘后确认，雇主电脑缺少 Visual C++ 运行库，导致程序在导出检测相关原生依赖加载时异常，表现为收费框无法触发或程序卡死。

**改动：**
- `build.spec` 显式收集 `msvcp140*.dll`、`vcruntime140*.dll`、`concrt140.dll`
- 后续交付必须使用最新 `build.bat` / `build.spec` 重新打包
- 文档中补充现场排查优先级：先确认是否运行最新 exe、是否缺 VC++ 运行库、是否被安全软件拦截

### 2. 启动说明强提示

**改动：**
- 启动后弹出操作说明提示框
- 现场人员必须点击“知道了”后，程序才进入监控流程
- 提示内容明确导出、付款、管理员密码解锁、再导出的正确顺序

### 3. 导出前全屏等待遮罩

**改动：**
- 新增“请等待 7 秒钟”全屏等待遮罩
- 点击像素蛋糕右上角黄色导出按钮后，程序尽快显示遮罩，阻止重复点击
- 新增 20ms 鼠标热区监听作为前置保护，不依赖截图 / OCR
- 保留监控线程 `export_button_pre_clicked` 作为兜底信号

**效果：** 慢电脑或非技术用户乱点场景下，程序能更早占住操作现场，给正式收费框检测争取时间。

## 2026-04-24 交付稳定性更新

本轮更新来自第二次远程交付后的问题复盘，并已在另一台 Windows 电脑上做兼容性测试，暂未发现明显异常。

### 1. 启动提示与壁纸

- `SoftwareUsageMeter.exe` 启动后会显示友好提示窗，说明程序已启动并正在等待像素蛋糕。
- 提示窗支持读取管理后台配置的壁纸，未配置时使用浅色兜底背景。
- 提示窗为非阻塞窗口，数秒后自动关闭，不影响后台监控启动。

### 2. 计时开始条件收紧

- 只有检测到可见、尺寸合理、标题有效的像素蛋糕主窗口后才开始计时。
- 仅存在后台残留进程、离屏窗口或异常最小化窗口时，不会提前计费。
- 像素蛋糕退出后，本轮计时直接结束并清零；再次启动像素蛋糕从 0 开始。

### 3. 收款后恢复与二次弹窗保护

- 进程挂起逻辑已做幂等处理，避免重复挂起造成付款后像素蛋糕卡死。
- 管理员确认收款后会恢复目标窗口交互，并记录恢复结果。
- 付款后的同一次导出尾迹会被保护，避免图片实际导出时再次触发收费框。

### 4. 现场日志

- exe 运行日志仍写入 exe 同目录 `app.log`。
- 启动早期异常会尝试写入 `%TEMP%\SoftwareUsageMeter_bootstrap.log`，用于排查状态页空白或启动无日志问题。

## 最新变更（本次迭代）

### 1. 放宽导出检测触发条件（核心改动）

**背景：** 雇主需在线下多台不同配置的电脑部署，原触发机制过于苛刻（视觉检测强制依赖 OCR 二次验证），导致部分机器无法正常弹出收费框。

**改动：**
- 视觉检测路径不再强制要求 OCR 验证，找到黄色导出按钮即可直接触发
- 导出按钮点击提升为独立强信号
- 居中对话框扫描默认开启，作为兜底检测手段
- OCR 仅负责识别导出张数，不再参与"是否弹窗"的决策

**效果：** 6 条检测路径（窗口标题 / 导出子进程 / 视觉按钮 / OCR 摘要 / 按钮点击 / 居中对话框扫描）任意一路命中即触发，显著提升多机兼容性。

**涉及文件：** `process_monitor.py`

### 2. 清理冗余文件

**删除内容：**
- 临时打包产物：`SoftwareUsageMeter.zip`、`package.bat`
- 诊断脚本：`diagnose_export.py`、`simple_diagnose.py`
- 测试脚本：`test_default_export.py`、`test_logic.py`、`test_ocr_debug.py`
- 测试目录：`tests/`（含编译缓存）
- 临时资源目录：`test_resources/`
- 误创建文件：`nul`
- Codex Plan 文件：`codex-plan-relax-export-detection.md`

**效果：** 项目结构精简，仅保留核心源码、配置与文档。

### 3. 文档全面更新

**更新文件：**
- `README.md` — 新增多路径检测说明，修复打包命令格式
- `开发说明.md` — 新增导出触发链路章节，更新项目结构、测试方法、排查建议
- `客户说明.md` — 简化 Q1 排查步骤，明确"即使 OCR 失败收费框仍会弹出"

## 已知限制与注意事项

1. **OCR 识别率**：Windows OCR 对部分分辨率/深色 UI 的识别效果不稳定，当前版本识别失败时要求管理员手动输入张数
2. **调试截图**：如需排查 OCR 问题，可在 `config.json` 中开启 `debug_export_capture: true`
3. **管理员密码**：首次部署**必须**修改默认密码 `admin`
